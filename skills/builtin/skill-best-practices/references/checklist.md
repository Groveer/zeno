# 有效 Skill 检查清单

在共享或发布技能前，请逐项核对，确保技能已达到必要质量标准。

## 必要项 (必须满足)

- [ ] **命名规范**：name 符合 64 字符限制，仅含小写字母/数字/连字符，无 XML 标签，无保留字（如 "anthropic", "agent"）。
- [ ] **描述规范**：description 格式正确（name/description），符合第三人称表述，包含功能和触发场景，不含 XML 标签。
- [ ] **长度限制**：SKILL.md 主文件内容少于 500 行，保持简洁。
- [ ] **引用结构**：所有文件引用采用一级引用深度，不嵌套。
- [ ] **路径规范**：所有文件路径使用正斜杠（/），禁止使用 Windows 风格路径。
- [ ] **术语一致**：整个文档库术语一致（使用通用 Agent 术语，避免平台特定称呼）。
- [ ] **无时间敏感信息**：文档中不包含会过期的日期或瞬时上下文。
- [ ] **清晰工作流**：所有流程步骤清晰、有逻辑，便于 Agent 遵循。
- [ ] **描述性命名**：所有文件和工具命名具有描述性，避免模糊（如 helper, utils）。
- [ ] **无保留字**：内容中不包含 anthropic、agent 等保留字（指明时除外）。

## 推荐项 (优化与测试)

- [ ] **多模型测试**：已使用不同层级的模型对技能进行评估。
- [ ] **真实场景评估**：至少创建了 3 个评估场景，且基于真实使用场景进行过测试。
- [ ] **团队反馈**：已纳入团队反馈（如适用）。
- [ ] **错误处理**：脚本包含明确的异常处理，不依赖 Agent 盲目尝试。
- [ ] **巫术常数优化**：所有配置值均有明确理由，代码中无不可解释的数字。
- [ ] **渐进式披露**：充分利用渐进式披露模式，SKILL.md 仅作概述，详细信息推送到参考文件。
- [ ] **脚本执行意图明确**：文档中明确区分了“脚本执行”与“代码参考”。
- [ ] **工具可发现性**：MCP 工具描述清晰，且不假设工具已预装。
