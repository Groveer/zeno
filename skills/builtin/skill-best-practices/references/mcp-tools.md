# MCP 工具

MCP（Model Context Protocol）工具允许 Agent 与外部系统、API 和数据库交互。通过 MCP，您可以将技能范围扩展到预定义的系统功能之外。

## 工具定位

确保工具描述清晰、简洁，并包含示例用法。Agent 可能无法定位工具，如果工具说明过于模糊。

### 良好工具描述的示例

```json
{
  "name": "query_database",
  "description": "对生产数据库执行 SQL 查询。仅适用于只读操作。使用此工具检索用户信息、交易历史或产品详细信息。",
  "inputSchema": { ... }
}
```

## 工具发现与可发现性

- **提供明确的描述**：描述应当清楚定义工具能做什么、何时应该调用它，以及它对 Agent 环境的预期影响。
- **避免过度加载**：不要定义过多的细粒度工具。将相关的 API 调用逻辑封装在更高级别的任务脚本中，并将其作为主要工具公开。
- **错误处理**：工具响应应包含结构化的错误信息，使 Agent 能清楚了解是哪一步调用失败了，以便其进行重试或回退操作。
